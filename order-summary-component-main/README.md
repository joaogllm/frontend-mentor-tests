## Overview

This is my solution to the [Order Summary Component challenge on Frontend Mentor](https://www.frontendmentor.io/challenges/order-summary-component-QlPmajDUj). It was a great exercise for practicing card layouts, flexbox alignment, hover states, and responsive design using only HTML and CSS.

### The challenge

Users should be able to:

- View the optimal layout for the component depending on their device's screen size
- See hover states for all interactive elements on the page

### AI Collaboration

This project was built with assistance from [Claude](https://claude.ai) (Anthropic) in the following areas:

- **CSS Reset** — Claude provided a modern, project-appropriate reset as a starting point
- **Code Review** — After completing the build, I submitted my HTML and CSS for a full review. Claude identified a critical bug (`.body` typo in the mobile media query), a missing `overflow: hidden` on the card container, and several improvements around accessibility (`:focus-visible` styles) and CSS maintainability (magic numbers, variable naming)

All design decisions, implementation, and final code were written by me. AI was used as a learning and feedback tool, not to generate the solution.
