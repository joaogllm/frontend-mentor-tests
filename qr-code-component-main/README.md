## Overview

This project is a solution to the QR Code Component challenge from Frontend Mentor. The goal of the challenge is to build a simple, responsive card component that closely matches the provided design, focusing on layout accuracy, typography, spacing, and visual hierarchy.

The component displays a QR code image along with a title and supporting text, centered both horizontally and vertically within the viewport. The project was built using semantic HTML and modern CSS, with an emphasis on clean structure, readability, and maintainability.

This challenge served as an introduction to Frontend Mentor projects and provided hands-on practice with:

Flexbox layout and centering techniques

Responsive sizing without fixed heights

Consistent spacing and typography

Writing clean, well-structured HTML and CSS

Deploying static projects using GitHub Pages

The final result is a lightweight, accessible, and visually accurate component that follows front-end best practices and establishes a solid foundation for more complex projects.

### AI Collaboration

🤖 Use of AI Tools

During this project, I used ChatGPT as a learning and support tool to improve both the quality of my code and my understanding of front-end fundamentals.

Tools used

ChatGPT – for guidance, explanations, and code review

How I used it

Asked questions about CSS layout concepts, especially Flexbox and centering elements correctly.

Requested explanations of best practices for HTML semantics and responsive CSS.

Used it to review my code and identify improvements related to accessibility, structure, and scalability.

Clarified small details such as color opacity using rgba(), spacing, and box-shadow values.

Verified my understanding of GitHub Pages deployment for hosting multiple projects in a single repository.

What worked well

Clear explanations helped me understand why certain solutions are better, not just what to write.

Code review feedback highlighted issues I wouldn’t have noticed on my own (e.g., unnecessary fixed sizes and semantic HTML improvements).

Helped me move faster without copying solutions blindly, reinforcing learning instead of replacing it.

What didn’t work / limitations

AI suggestions still required critical thinking and manual testing.

Some recommendations needed adjustment to match the exact Frontend Mentor design specifications.

Visual accuracy and final refinements depended on my own inspection and comparison with the design files.

Overall takeaway

ChatGPT was used as an assistant, not a replacement for learning. It supported my problem-solving process, improved my understanding of front-end concepts, and helped me write cleaner, more maintainable code while keeping full ownership of the final implementation.
